# Trade Grading Rubric — เกณฑ์ให้เกรด Trade (A/B/C/D)

ทุกครั้งที่ระบบจะเปิด trade (ผ่าน Risk Agent แล้ว) จะคำนวณ "เกรด" ของ trade นั้นโดยอัตโนมัติ
จากข้อมูลที่ agent ทุกตัววิเคราะห์ไว้แล้ว — **ไม่เรียก LLM เพิ่ม** จึงไม่มีค่าใช้จ่ายหรือ latency เพิ่ม

เป้าหมาย: เก็บเกรดไว้กับทุก trade ใน DB (`trades.grade`) + แสดงใน dashboard
เพื่อในอนาคตดูได้ว่า win rate ของแต่ละเกรดต่างกันแค่ไหน ก่อนตัดสินใจว่าจะ filter เฉพาะเกรดดีๆ หรือไม่

ดู implementation ที่ `core/trade_grader.py`

---

## องค์ประกอบ (3 ปัจจัย, รวม 3-9 คะแนน)

### 1. Consensus Weight Ratio
น้ำหนักรวมของ agent ฝั่งที่ชนะ หาร น้ำหนักรวมทั้งหมด (9.5)
ต้อง ≥ 40% อยู่แล้วถึงจะผ่าน master_agent (`MIN_WEIGHT_RATIO`)

| Ratio | คะแนน | ความหมาย |
|-------|-------|----------|
| ≥ 70% | 3 | agent ส่วนใหญ่เห็นตรงกันชัดเจน |
| ≥ 55% | 2 | เสียงข้างมากชัดเจน |
| ≥ 40% | 1 | เพิ่งผ่านเกณฑ์ขั้นต่ำ |

### 2. Signal Strength
ค่าสัมบูรณ์ของ weighted total score เทียบกับ threshold (±5.0)

| \|Score\| | คะแนน | ความหมาย |
|-----------|-------|----------|
| ≥ 15 | 3 | สัญญาณแรงมาก |
| ≥ 8  | 2 | สัญญาณชัดเจน |
| < 8  | 1 | เพิ่งผ่าน threshold หรือเป็น grey-zone (LLM ตัดสิน) |

### 3. Confidence
master_confidence = ค่า confidence สูงสุดของ agent ทั้งหมด
ต้อง ≥ 60% อยู่แล้วถึงจะผ่าน risk_agent (`MIN_CONFIDENCE`)

| Confidence | คะแนน |
|------------|-------|
| ≥ 85% | 3 |
| ≥ 70% | 2 |
| ≥ 60% | 1 |

---

## รวมคะแนน → เกรด

| รวม (3-9) | เกรด |
|-----------|------|
| 8-9 | A |
| 6-7 | B |
| 4-5 | C |
| 3   | D |

## Volatility Modifier

ถ้า ATR/price < 0.5% (เพิ่งผ่านเกณฑ์ขั้นต่ำ `MIN_ATR_RATIO=0.003` ของ risk_agent) → **ลดเกรดลง 1 ขั้น** (ต่ำสุด D)

เหตุผล: ตลาดนิ่งเกินไป TP/SL ที่คำนวณจาก ATR จะแคบมาก เสี่ยงโดน noise สวิงชน SL ก่อนถึง TP

---

## วิธีใช้ (Phase ปัจจุบัน)

- คำนวณเกรดอัตโนมัติตอนเปิด trade ทุกไม้ → เก็บใน `trades.grade`
- แสดงใน dashboard (คอลัมน์ Grade ใน Trade History)
- **ยังไม่ filter ตามเกรด** — รอสะสม closed trades ~15-20 ไม้ก่อน ค่อยดู win rate แยกตามเกรดจริงจากข้อมูล

## Phase ถัดไป (บันทึกใน PROGRESS.md)

- สรุป win rate แยกตามเกรด → feed กลับเข้า prompt ของ master_agent (LLM) เป็น "memory"
- เมื่อข้อมูลพอ (~15-20 closed trades) → พิจารณา filter เทรดเฉพาะเกรด A (หรือ A/B) เท่านั้น
